# Sample PRD for AI Coding

## 1. Overview

### Feature Name

顧客一覧ページに検索機能を追加する

### Background

今の顧客一覧ページは、表示されたデータを目視で探すしかありません。  
件数が増えると、目的の顧客を見つけるのに時間がかかります。

### Goal

ユーザーが名前やメールアドレスの一部を入力すると、該当する顧客だけを一覧表示できるようにする。

## 2. Problem

- 顧客が多いと探しづらい
- ジュニア開発者が実装するときに、どこまで作ればよいか分かりにくい
- AI に依頼するときに、要件が曖昧だと不要な実装が増えやすい

## 3. Scope

### In Scope

- 顧客一覧ページに検索入力欄を追加する
- 名前とメールアドレスを対象に部分一致検索を行う
- 入力中に一覧が更新される
- 検索結果が 0 件のときの表示を出す

### Out of Scope

- 高度な絞り込み
- 並び替え機能
- サーバーサイド検索
- ページネーション改善

## 4. Target User

- 顧客一覧を使う社内メンバー
- このテンプレートを学習中のジュニア開発者
- PRD を読んで実装する AI コーディングエージェント

## 5. User Stories

- ユーザーとして、顧客名の一部で検索したい。目的の顧客を早く見つけたいから。
- ユーザーとして、メールアドレスの一部でも検索したい。顧客情報を特定しやすくしたいから。
- 開発者として、要件が明確な PRD を使いたい。AI に実装を依頼しやすくしたいから。

## 6. Functional Requirements

| ID | Requirement |
|---|---|
| FR-1 | 顧客一覧ページの上部に検索入力欄を表示する |
| FR-2 | 入力値に応じて一覧を絞り込む |
| FR-3 | 検索対象は `name` と `email` にする |
| FR-4 | 大文字小文字の違いは無視する |
| FR-5 | 0 件のときは空状態メッセージを表示する |
| FR-6 | 入力を空にすると全件表示に戻る |

## 7. Non-Functional Requirements

- 実装は既存 UI の見た目を大きく崩さない
- 複雑な状態管理を追加しない
- 読みやすいコードにする
- 既存の型エラーを増やさない

## 8. Acceptance Criteria

1. 顧客一覧ページに検索欄が表示される
2. `tanaka` と入力すると、名前またはメールに `tanaka` を含む顧客だけが表示される
3. `TANAKA` のように大文字で入力しても同じ結果になる
4. 一致する顧客がいない場合、空状態メッセージが表示される
5. 入力を削除すると元の一覧に戻る

## 9. Edge Cases

- 前後に空白が入っている
- 空文字が入力されている
- 同じ文字列が名前とメールの両方に含まれる
- 顧客データが 0 件の状態

## 10. Implementation Notes for AI Agents

### Expected Files

- `app/pages/customers.vue`
- 必要なら `app/components/` 内の関連 UI

### Constraints

- 既存レイアウトをできるだけ維持する
- 新しいライブラリは追加しない
- まずはクライアントサイドで完結させる

### Suggested Steps

1. 顧客一覧ページの現在の構成を確認する
2. 検索入力の UI を追加する
3. 絞り込みロジックを実装する
4. 0 件表示を追加する
5. 型チェックを実行する

### Definition of Done

- Acceptance Criteria をすべて満たす
- 型チェックで致命的なエラーが出ない
- PRD を読んだ別の開発者が、目的と範囲を理解できる
